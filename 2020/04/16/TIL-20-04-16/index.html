05. 제이쿼리 모바일 기초 다지기

  • DOM(Document Object Model) 참고하기
    • DOM 소개(from 모질라) : https://developer.mozilla.org/ko/docs/Web/API/Document_Object_Model/%EC%86%8C%EA%B0%9C
    • https://wit.nts-corp.com/2019/02/14/5522
    • 브라우저는 어떻게 동작하는가? : https://d2.naver.com/helloworld/59361

  • 웹 애플리케이션 개발 동향

    • 애플리케이션(데스크톱)
    • 웹애플리케이션(데스크톱)
    • 모바일 웹 애플리케이션(웹앱)
    • 모바일 애플리케이션(네이티브앱)

  • 모바일 환경에서는 사용자 경험이 강조됨

  • 모바일 UI 프레임워크

    • 제이쿼리 모바일(실질적으로 가장 많이 사용됨) - jQuery를 확장한 프레임워크
    • 센차 터치
    • 제이큐터치

  • 제이쿼리 모바일

    • 장점
      • 제이쿼리 모바일 확장 속성만 활용, 제이쿼리 지식이 부족해도 모바일 화면을 구성 가능
      • 거의 모든 모바일 장치를 지원
      • CSS3로 만들어진 여러 테마를 조합하여 세련된 디자인이 가능
    • 단점
      • 제이쿼리와 제이쿼리 모바일 라이브러리를 추가로 필요로 하므로 용량이 커져 대역폭과 로딩시간이 늘어남

  • 프레임워크 연결 방식

    • CDN 방식(컨텐츠 전송 네트워크)
      • 파일을 직접 다운받지 않고 제이쿼리 모바일 서버의 URL 주소만을 마크업에 명세함으로써 프레임워크가 적용될 때 참조하는 방식
      • 인터넷 연결이 필수적임
    • 다운로드 방식

  • 제이쿼리 모바일 페이지 구조

    • 페이지 - 브라우저에 보여지는 전체 화면 영역, 헤더, 콘텐츠, 푸터를 포함하는 컨테이너 역할
    • 헤더
    • 콘텐츠
    • 푸터

  • 모바일 페이지 구성 유형

    • 단일페이지 탬플릿
      • HTML5 문서 안에 오직 하나의 페이지를 정의
    • 다중 페이지 탬플릿
      • HTML5 문서 안에 둘 이상의 페이지를 정의
      • 초기에 모든 페이지를 읽어 들이므로 부담이 되지만 이 후에는 페이지 연결 속도가 매우 빠르다.

  • 페이지 연결 방식

    • 내부 페이지 연결 - 하나의 HTML5 파일 안에 여러 페이지를 명세하여 서로 연결하는 방법
    • 외부 페이지 연결 - 여러 HTML5파일 안에 페이지들을 나누어 명세하고 서로 연결하는 방법

  • 페이지 전환 효과

    • 제이쿼리 모바일은 CSS3를 사용, 페이지 이동 과정에서 화면 전환 효과를 제공

06. 제이쿼리 모바일 헤더바와 푸터바

1. 헤더바

  • 헤더 문자열 추가
    • 헤더 영역 구분: 왼쪽 영역, 제목 문자열 영역, 오른쪽 영역
    • 헤더바의 문자열은 주로 페이지 제목을 표시(h1~h6사용)
  • 헤더 버튼 추가

2. 푸터바

  • 푸터바
  • 푸터바 문자열 추가
  • 푸터바 버튼 추가

4. 네비게이션바

  • 헤더바의 경우, 3개 이상의 버튼이 필요할 경우 네비게이션바를 사용

  • 보통 모바일 기기를 손에 쥐고 쉽게 메뉴를 선택하도록 푸터바에 많이 배치

  • 네비게이션 바 추가

  • 네비게이션바 버튼 활성화

  • 헤더바와 푸터바의 고정 배치

    • 헤더바와 푸터바의 표시 위치를 화면 맨 위와 맨 밑으로 고정

  • 헤더바와 푸터바의 숨김

    • 전체화면 모드 설정
    • 헤더바와 푸터바를 모바일 페이지에서 사라지게 하고 필요할 경우에만 화면에 동적으로 표시
    • 그림, 사진, 비디오 뷰어 등에서 미디어 내용을 화면 전체에 나타내고자 할 때
    • 화면을 터치함으로써 헤더바와 푸터바의 표시와 숨김 두가지 상태를 반복

5. 툴바

  • 지속형 푸터바 설정
    • ​ 여러 페이지의 푸터바가 모두 동일할 경우, 페이지 이동 시에도 화면 하단에 고정되어 표시 가능
